Startup Settings

Basic Information

  • Every framework requires a platform, device, and path to your Soda scripts project. Some frameworks require more information...
  • The "Platform", "Device", and "Path to Project" fields are all required.
  • The "Platform" is the type of device. For Instruments, the options are iPhone and iPad. For Automator: Android Tablet and Android Phone. For web: it's always just "web".
  • The "Device" is the simulator/emulator/physical device you wish to start Soda on. These fields are pre-populated by Soda based on what's currently available on the server.
  • If you wish to start Soda on a physical device, ensure it's plugged in before launching this page (or you'll have to refresh the page).
  • The "Path to Project" is the path to your Soda project. It must contain the "soda.json" file.
  • So you may be wondering why you have to chose a platform, when clearly you've chosen a simulator that is specific to a certain platform (i.e. iPhone). This is because, The platform option is used to determine which non-generic test scripts to use. This allows you to change the platform and use different scripts on different devices.
  • Use the "environment" field to specify a testing environment

Instruments Specific Settings

  • Path to App: Required. The path to the binary for the app you'd like to start Soda on.

Automator Specific Settings

  • Path to APK: Required. The path to the .APK file for the app you'd like to start Soda on.
  • Proxy URL: Optional. An optional proxy url (if behind a corporate proxy)
  • Proxy Username/Password: Optional. For use in proxy authentication

Web Specific Settings

  • Starting URL: Optional. The inital web page to navigate to. If none is specified, "about:blank" will be used.
  • Proxy URL: Optional. An optional proxy url (if behind a corporate proxy)
  • Proxy Username/Password: Optional. For use in proxy authentication

Framework Configuration

Instruments Configuration

  • The Instruments framework requires no real setup—Apple makes it fairly easy. However, if you're having problems starting up check the settings defined in Soda/SodaCore/frameworks/instruments/imports/Config.js.
  • Soda queries Instruments to find all available devices, if the desired device isn't listed... read on.
  • Can't find the right simulator version? Run the command: sudo xcode-select --switch /path/to/XCodeIWantToUse.app to switch XCode binaries.
    Once the desired version is selected, refresh the page and the available device list will update.

Automator Configuration

  • If you're having issues, ensure that the environment variable ANDROID_SDK_HOME is set and points the the path where you've installed the Android SDK tools.
  • Soda searches the file system to find all available ADB devices, if the desired device isn't listed, there is likely a problem with your ADB device. Try removing and re-creating it, then refreshing the page.
  • Since ADB Emulators take *forever* to startup, Soda makes use of attaching to running emulators. For faster startup, leave the emulator running between sessions, if possible.

Web Configuration

  • Chrome Driver must be installed for the Selenium framework to work with Google Chrome!
